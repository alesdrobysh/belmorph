# Архітэктурны Дызайн-дакумент: Бібліятэка `belaz` (Беларуская азбука)

## 1. Кантэкст і Матывацыя
На дадзены момант у экасістэме JavaScript/Node.js не існуе паўнавартасных, аўтаномных марфалагічных аналізатараў беларускай мовы. Распрацоўшчыкі вымушаныя пісаць уласныя "кастылі" на рэгулярных выразах або рабіць павольныя сеткавыя запыты да Python-сервераў (напрыклад, з `pymorphy2`). 

Мэта **`belaz`** — стварыць хуткі, кампактны марфалагічны рухавік, які працуе як у Node.js, так і ў браўзеры (offline-first). База дадзеных генеруецца на аснове акадэмічных дадзеных праекта **GrammarDB** (Беларускі N-корпус). 

Назва — гэта адсылка да магутнага беларускага самазвала БелАЗ (бо бібліятэка цягне на сабе сотні тысяч слоў) і бэкронім "Беларуская азбука". Гэта ідэйны працяг папулярнай рускамоўнай бібліятэкі `az.js`.

## 2. Мэты і Абмежаванні (Goals & Non-Goals)

### ✅ Мэты (Што мы падтрымліваем):
* **Марфалагічны аналіз:** Вызначэнне часціны мовы, склону, роду, ліку і г.д. па ўведзеным слове.
* **Скланенне і ўтварэнне формаў (Inflection):** Генерацыя патрэбнай формы (напрыклад, *назоўнік, творны склон, множны лік*).
* **Лематызацыя:** Звядзенне слова да пачатковай (слоўнікавай) формы.
* **Прадказанне (Predictor):** Здольнасць апрацоўваць незнаёмыя (выдуманыя ці новыя) словы на аснове іх суфіксаў і канчаткаў.
* **Ізаляванасць:** Нуль знешніх залежнасцяў (zero-dependency) у канчатковым NPM-пакеце.

### ❌ Абмежаванні (Чаго мы НЕ робім):
* **Сінтаксічны аналіз:** Бібліятэка не аналізуе сказы цалкам і не будуе сінтаксічныя дрэвы.
* **Кантэкстная дызамбігуацыя:** Калі слова мае дзве розныя пачатковыя формы (напрыклад, "маці" — і назоўны, і родны склон), `belaz` вяртае абодва варыянты. Угадванне па кантэксце застаецца на баку распрацоўшчыка.

## 3. Высокаўзроўневая Архітэктура

Сістэма лагічна падзелена на дзве цалкам ізаляваныя часткі: **Builder (Зборшчык)** і **Runtime Client (Кліенцкі пакет)**.



### Частка А: Builder (Скрыпты падрыхтоўкі дадзеных)
Гэта скрыпты (Node.js), якія выкарыстоўваюцца толькі падчас распрацоўкі і зборкі новай версіі слоўніка.
1. **Extract (Здабыванне):** Чытае зыходную базу `grammardb.sqlite3` (240k+ лем).
2. **Analyze (Аналіз):** Знаходзіць асновы (stems) слоў і збірае ўсе ўнікальныя схемы змянення ў **Парадыгмы**.
3. **Compress (Сцісканне):** Будуе структуру DAWG (Directed Acyclic Word Graph) для максімальнага сціскання слоўнікавай базы.
4. **Export (Экспарт):** Генеруе аптымізаваныя файлы `paradigms.json` і `dict.dawg`.

### Частка Б: Runtime Client (Кліенцкі пакет)
Гэта тое, што карыстальнікі спампоўваюць праз `npm install belaz`. Змяшчае толькі "рухавік" (engine) для хуткага чытання графа і прымянення парадыгмаў.

## 4. Схема Дадзеных у Runtime

Каб забяспечыць мінімальны памер і максімальную хуткасць, лінгвістычныя дадзеныя захоўваюцца ў двух фарматах:

| Кампанент | Структура | Апісанне |
| :--- | :--- | :--- |
| **Парадыгмы** | Масіў аб'ектаў | Індэкс масіва = `Paradigm ID`. Змяшчае правілы. Прыклад: `[ [ {tag: 'NOM_SG', suf: 'а'}, {tag: 'INS_PL', suf: 'амі'} ... ] ]` |
| **Слоўнік (DAWG)** | Прэфікснае дрэва | Вузлы графа. Пошук слова `горад` ідзе па галінках: `г -> о -> р -> а -> д`. На літары `д` вісіць ідэнтыфікатар парадыгмы. |

## 5. Дызайн API

API распрацаваны так, каб быць інтуітыўным і сумяшчальным з сучаснымі ES6 модулямі.

```javascript
import { MorphAnalyzer } from 'belaz';

// Ініцыялізацыя (загружае DAWG-слоўнік у памяць)
const morph = new MorphAnalyzer();

// 1. Аналіз слова (Parsing)
const parsed = morph.parse('гарадамі');
console.log(parsed[0].lemma);       // "горад"
console.log(parsed[0].tags.case);   // "ins" (творны)
console.log(parsed[0].tags.number); // "plur" (множны лік)

// 2. Утварэнне формы (Inflection)
const city = morph.parse('горад')[0];
const pluralInst = city.inflect({ number: 'plur', case: 'ins' });
console.log(pluralInst.word); // "гарадамі"

// 3. Вывад усіх формаў
console.log(city.lexeme); // Спіс усіх магчымых формаў слова "горад" з тэгамі
