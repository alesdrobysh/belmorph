# belmorph

[English](README.md)

**Бібліятэка марфалагічнага аналізу беларускай мовы без залежнасцяў**

Хуткая, лёгкая бібліятэка для аналізу беларускіх слоў і іх марфалагічных уласцівасцяў.

## Магчымасці

- **Марфалагічны аналіз**: Вызначэнне часціны мовы, склону, роду, ліку, часу і іншых граматычных уласцівасцяў
- **Скланенне**: Генерацыя розных граматычных форм слоў
- **Генерацыя лексемы**: Атрыманне ўсіх магчымых форм слова
- **Марфалагічнае прадказанне**: Невядомыя словы аналізуюцца па суфіксальных узорах
- **Без залежнасцяў**: Ніякіх знешніх залежнасцяў пры выкананні
- **Падтрымка TypeScript**: Поўныя вызначэнні тыпаў
- **Эфектыўнае захоўванне**: Выкарыстоўвае DAWG (Directed Acyclic Word Graph) для хуткага пошуку
- **Сумяшчальнасць з браўзерам**: Працуе ў Node.js, браўзерах і Deno — без native залежнасцяў
- Слоўнік ахоплівае 6 574 парадыгмы скланення, атрыманыя з GrammarDB

## Усталёўка

```bash
npm install belmorph
```

## Хуткі старт

```typescript
import { MorphAnalyzer } from 'belmorph';

const morph = new MorphAnalyzer();
const results = morph.parse('горад');

console.log(results[0].lemma);    // 'горад'
console.log(results[0].tags.pos); // 'N' (назоўнік)

// Скланенне да патрэбнай формы
results[0].inflect({ case: 'I', number: 'P' })?.word; // 'гарадамі'

// Поўныя назвы і кароткія коды ўзаемазамяняльныя
results[0].inflect({ case: 'instrumental', number: 'plural' })?.word; // 'гарадамі'
results[0].inflect({ case: 'I', number: 'plural' })?.word;            // той жа вынік

// Атрымаць усе формы
const lexeme = results[0].lexeme;
console.log(lexeme.map(r => r.word));
```

### Прыклад з дзеясловам

```typescript
const results = morph.parse('пісаць');
console.log(results[0].lemma);    // 'пісаць'
console.log(results[0].tags.pos); // 'V' (дзеяслоў)
```

### Прыклад з прыметнікам

```typescript
const results = morph.parse('вялікі');
console.log(results[0].lemma);    // 'вялікі'
console.log(results[0].tags.pos); // 'A' (прыметнік)
```

## API

### MorphAnalyzer

Асноўны клас для марфалагічнага аналізу.

```typescript
const morph = new MorphAnalyzer(dictPath?: string);
```

Параметр `dictPath` неабавязковы і па змаўчанні паказвае на ўбудаваную дырэкторыю `dict/`. Звычайна яго не трэба перадаваць.

- `parse(word: string): ParseResult[]` — разбор слова і вяртанне ўсіх магчымых аналізаў

### ParseResult

Вынік марфалагічнага аналізу.

Уласцівасці:
- `word: string` — форма слова
- `lemma: string` — слоўнікавая форма
- `tags: Grammeme` — граматычныя ўласцівасці
- `predicted: boolean` — ці быў аналіз прадказаны (слова няма ў слоўніку)

Калі слова не знойдзена ў слоўніку, аналізатар выкарыстоўвае прадказанне на аснове суфіксаў. Такія вынікі маюць `predicted: true` і могуць быць менш дакладнымі:

```typescript
if (results[0].predicted) {
  // Аналіз заснаваны на суфіксальных узорах, можа быць менш дакладным
}
```

Метады:
- `inflect(target: Partial<GrammemeInput>): ParseResult | null` — скланенне да патрэбнай формы
- `get lexeme(): ParseResult[]` — атрыманне ўсіх форм слова

### Grammeme

Інтэрфейс для граматычных уласцівасцяў:

```typescript
interface Grammeme {
  pos?: Pos;        // Часціна мовы
  case?: Case;      // Склон
  gender?: Gender;  // Род
  number?: Num;     // Лік
  person?: Person;  // Асоба
  tense?: Tense;    // Час
  mood?: Mood;      // Лад
}
```

Метад `inflect()` прымае як кароткія коды, так і поўныя назвы на англійскай мове для склону, роду, ліку, часу і ладу (гл. табліцы ніжэй).

#### Pos (Часціна мовы)

| Значэнне | Апісанне |
|----------|----------|
| `'N'` | Назоўнік |
| `'A'` | Прыметнік |
| `'V'` | Дзеяслоў |
| `'E'` | Прыслоўе |
| `'P'` | Займеннік |
| `'C'` | Злучнік |
| `'I'` | Выклічнік |
| `'Z'` | Часціца |

#### Case (Склон)

| Значэнне | Поўная назва | Апісанне |
|----------|--------------|----------|
| `'N'` | `'nominative'` | Назоўны склон |
| `'G'` | `'genitive'` | Родны склон |
| `'D'` | `'dative'` | Давальны склон |
| `'A'` | `'accusative'` | Вінавальны склон |
| `'I'` | `'instrumental'` | Творны склон |
| `'L'` | `'locative'` | Месны склон |
| `'V'` | `'vocative'` | Клічны склон |

#### Gender (Род)

| Значэнне | Поўная назва | Апісанне |
|----------|--------------|----------|
| `'M'` | `'masculine'` | Мужчынскі род |
| `'F'` | `'feminine'` | Жаночы род |
| `'N'` | `'neuter'` | Ніякі род |

#### Num (Лік)

| Значэнне | Поўная назва | Апісанне |
|----------|--------------|----------|
| `'S'` | `'singular'` | Адзіночны лік |
| `'P'` | `'plural'` | Множны лік |

#### Person (Асоба)

| Значэнне | Апісанне |
|----------|----------|
| `'1'` | Першая асоба |
| `'2'` | Другая асоба |
| `'3'` | Трэцяя асоба |

#### Tense (Час)

| Значэнне | Поўная назва | Апісанне |
|----------|--------------|----------|
| `'R'` | `'present'` | Цяперашні час |
| `'P'` | `'past'` | Прошлы час |
| `'F'` | `'future'` | Будучы час |

#### Mood (Лад)

| Значэнне | Поўная назва | Апісанне |
|----------|--------------|----------|
| `'I'` | `'indicative'` | Абвесны лад |
| `'M'` | `'imperative'` | Загадны лад |

## Зборка слоўніка

Бібліятэка патрабуе загадзя зробленага слоўніка. Для яго зборкі:

1. Упэўніцеся, што субмодуль `GrammarDB` ініцыялізаваны:

```bash
git submodule update --init
```

2. Запусціце зборку слоўніка:

```bash
npm run build:dict
```

Гэта створыць неабходныя файлы слоўніка ў дырэкторыі `dict/`.
